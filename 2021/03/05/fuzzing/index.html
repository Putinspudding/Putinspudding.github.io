VxWorks_Fuzzing

发表于 2021-03-05 更新于 2021-03-06

  首先在虚拟机中打开iso镜像安装VxWorks5.5与6.8，接着修改VM的物理地址使其和License中第二行最后的物理地址相同，修改BSP时，将X:\WindRiver\xvworks-6.8\target\config\pcPentium4复制一份命名为Pentium4,放入pcPentium4同一文件夹中，之后是对Pentium4文件夹中的文件进行更改，可以使用用编辑器对配置文件进行修改，也可以使用已经修改好的文件,将其复制覆盖放入X:\WindRiver\xvworks-6.8\target\config\Pentium4中，对X:\WindRiver\xvworks-6.8\target\config\Pentium4\config.h 可能还需要后续更改。

  打开WindRiver,创建Boot Loader/BSP Project,右键Build之后shell打开，cd Pentium4，make bootrom.bin，将WindRiver\workspace\xxxx(BSP Project 名)\bootrom.bin 重命名成 bootrom.sys 并提取出来存放在显眼位置。找寻WindRiver\vxworks-6.8\host\x86-win32\bin下的vxload.com和vxsys.com两个文件，将其与bootrom.sys放在一块,创建 VxWorks Image Project ,以刚刚创建的 BSP Project 为基底，右键 Build Image Project，找到 WindRiver\workspace\xxxx(Image Project 名)\default\vxWorks，也将其复制存放在一个单独目录。

  新建一个虚拟机，使用软盘镜像，进入DOS系统，Other IDE CD-ROM->输入 fdisk 来进行硬盘分区 ->Create DOS partition or Logical DOS Drive->Create Primary DOS partition->输入Y->系统自动重启->Other IDE CD-ROM->输入 format c:\s 对C盘格式化->输入Y->回车。完成之后就要对虚拟机进行映射了， 映射按钮是灰色的，查阅官网文档发现，独立版本的 Workstation Player 不支持将虚拟磁盘映射到主机系统。Workstation Pro 随附的 Workstation Player 版本则支持虚拟磁盘映射。又去下载安装了个Workstation Pro，可以对虚拟机进行映射了。将bootrom.sys，vxload.com和vxsys.com三个文件放入映射文件夹后，打开DOS虚拟机，进行如下操作：输入 c:->输入vxsys c:->按Y确认，关闭虚拟机，取消勾选软盘的启动时连接，再次开启虚拟机。结果如下图所示： 原以为这是config.h 中 ip 地址配错的原因，调试了好半天 ip 地址与 config.h 最后发现是在config.h 中 host:vxWorks 之间多了一个空格导致运行错误，删除空格后的结果： 查找发现可能是文件内部配置有问题，用上文给的已经修改好的文件覆盖再重新执行上述步骤之后， 打开WindRiver\vxworks-6.8\host\x86-win32\bin中的wftpd32.exe，设置用户名密码都为target，主目录为vxWorks所在目录。设置wftpd32的Log Options,除了“Winsock Calls”不勾选，其他一律选上。重启VxWorks虚拟机。会出现两种情况， 一种：wftpd32无任何log，VxWorks启动后命令栏中只有[VxWorks Boot]。这种的主要原因是config设置IP与主机IP不一致造成的。更改config/更改主机IP任一方法都能解决此问题；第二种，即上图这种，表明wftpd32主目录设置出错,vxWorks不在主目录中，更改wftpd32的主目录，结果如下：
